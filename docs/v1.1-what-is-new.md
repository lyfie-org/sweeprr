# What's New in Sweeprr v1.1

Sweeprr v1.1 is a major feature release spanning five sprints of work (Sprints 7–11). It adds
deeper Jellyfin integration, smarter cleanup rules, curation tooling, security features, and
operational safeguards like automated backups and alerting.

This page is a narrative tour of what changed. For step-by-step setup of specific features, see
the linked guides in each section.

---

## Smarter, More Reliable Rules

- **Playstate now persists in the database.** Watched/in-progress status survives restarts —
  no more re-learning playback state from Jellyfin on every boot.
- **Genre and resolution conditions are live.** Rules can match on a movie or episode's genre
  list and video resolution (e.g. target 480p re-encodes for cleanup, or exclude a favorite genre).
- **New TV-lifecycle conditions**: `SeriesEnded`, `IsSeasonFinale`, and `CutoffMet` (whether the
  episode has reached its quality profile's cutoff) let you build rules like "delete watched
  episodes of ended shows that have already hit their quality cutoff."

---

## Action Escalation & Smarter Exclusions

- **Quality profile downgrades** (`ChangeQualityProfile`) are now a sweep action — instead of
  deleting a large 4K file outright, Sweeprr can ask Radarr/Sonarr to downgrade it to a smaller
  profile and let the *arr apps handle the re-grab.
- **Scoped and tag-based exclusions.** Exclusions can now be limited to specific rule groups, and
  entire Radarr/Sonarr tags can be whitelisted from sweeps in one shot.
- **Direct Jellyfin deletion fallback.** Orphaned Jellyfin-only media (items with no matching
  Radarr/Sonarr record) can optionally be deleted directly via the Jellyfin API. This is
  off by default — enable it deliberately in Settings, since it bypasses the *arr apps entirely.
- **Dual-instance *arr awareness.** If you run more than one Radarr or Sonarr instance, Sweeprr
  detects when the same title exists across instances so cleanup rules don't act on a copy
  that's still needed elsewhere.
- **Bazarr subtitle cleanup.** When Sweeprr removes a movie or series, it now asks Bazarr to
  clean up the associated subtitle files too. See [Bazarr Integration](bazarr-integration.md).

---

## "Leaving Soon" Curation & Visual Tools

- **Native Jellyfin collection sync.** A "Sweeprr - Leaving Soon" collection is automatically
  kept in sync with your sweep queue, so users can browse what's about to be removed directly
  inside Jellyfin.
- **Poster overlay banners.** Optionally, Sweeprr stamps a "Leaving Soon" gradient banner
  directly onto poster art for queued items — and restores the original artwork automatically
  when an item is excluded, fails, or leaves the queue.
- **Disk-space-aware rules.** New rule fields (`DiskFreeSpacePercent`, `DiskFreeSpaceGb`) let
  you build rules that only trigger cleanup when a library's underlying disk is running low —
  e.g. "only delete watched movies if free space drops below 10%."
- **Sandbox simulator.** The new Sandbox view lets you preview exactly what a rule group would
  do — and how much space it would reclaim — before enabling it for real.
- **Media Explorer / Curation Dashboard.** A new "Media" tab gives a searchable, filterable view
  of your entire library with at-a-glance status (matched rules, queue state, disk usage) —
  no more guessing what a sweep will touch.

---

## Deeper Jellyfin Integration & Security

- **Jellyfin Playback Reporting plugin support.** If the Playback Reporting plugin is installed,
  Sweeprr can pull richer watch-history data for more accurate "last watched" rules.
- **Session control & in-app alerts.** Active Jellyfin sessions now receive an in-app warning
  before a scheduled sweep runs, and again if the item they're currently watching is queued for
  removal.
- **Scoped API keys.** Generate `spr_live_...` API keys with fine-grained scopes
  (`read:sweep`, `write:sweep`, `execute:sweep`, `admin`) for automation and integrations,
  without sharing admin login credentials. See [API Keys](api-keys.md).
- **"Request Extension" public portal.** End users can sign in with their own Jellyfin
  credentials at `/extend` and request more time for an item before it's swept — no Sweeprr
  account needed. See [Extension Portal](extension-portal.md).
- **In-UI "Leaving Soon" banners.** A small script injected into Jellyfin's web client shows
  removal countdowns and an extension link directly on the item page. See
  [Jellyfin Script Injection](jellyfin-script-injection.md).

## Alerts, Portability & Backups

- **Discord & generic webhook notifications.** Get notified on sweep completion, failsafe
  trips, pending-item buildup, or connection errors — to Discord (rich embeds) or any
  JSON webhook endpoint.
- **Rule group import/export.** Export any rule group to a portable JSON file and import it
  into another Sweeprr instance — handy for sharing rule templates or migrating between
  environments. Imported groups are created disabled so you can review environment-specific
  values (paths, tags) before turning them on.
- **Automated backups to Local or S3/MinIO.** Schedule recurring snapshots of the database and
  encryption keys, with configurable retention, or trigger a backup on demand. See
  [S3 / MinIO Backups](s3-backup.md).

---

## Upgrading

v1.1 ships its own EF Core migrations, which are applied automatically on startup — no manual
steps are required beyond pulling the new image. As always, take a backup before upgrading; see
[Backup & Restore](backup-restore.md) or set up the new [automated backups](s3-backup.md).
